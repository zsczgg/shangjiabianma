# 商品资料编辑与历史记录设计

商品内部编码在 SKU 创建时生成，此后作为不可变身份标识。编辑页面只读展示内部编码，更新接口的请求模型不包含该字段，数据库更新语句也永不写入该列。

商品名称、品牌、分类、备注、规格名称、厂家条码、菜鸟货品编码和平台商品关系均可修改。每次实际发生变化时，系统写入一条变更记录，包含商品、可选 SKU、字段名称、旧值、新值和发生时间。外部编码修改采用替换方式：旧记录从当前编码映射中删除，因此扫码与搜索立即失效；旧值仅存在于变更记录中供人工追溯。

保存操作在数据库事务中完成。若新外部编码已经属于另一 SKU，唯一约束会阻止保存并提示冲突，避免一个码对应多个商品。页面保存成功后返回商品详情，并在历史区域按时间倒序显示变化。
